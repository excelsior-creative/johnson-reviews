### One distinct technical contribution: EXIF GPS stripper

The mandate says: "always strip location metadata from photos that
include the kids before publishing." None of the seven open PRs
ship that capability. Tonight I built it as
`apps/app/scripts/strip-exif-gps.ts`:

- CLI script run as `pnpm strip:exif <path>` (added to
  `apps/app/package.json` scripts).
- Walks a path or directory recursively, finds JPEG/PNG/WebP/AVIF/TIFF.
- Detects GPS by scanning the EXIF buffer for the `0x8825` IFD tag.
- When GPS is found, re-encodes the image via `sharp` (already a
  dependency) — sharp's default drops all EXIF, which is the
  privacy-conservative behavior. Other photos (no GPS detected) are
  left untouched, preserving camera/lens/date metadata for E-E-A-T.
- Atomic write: temp file then rename. Failures don't corrupt the
  original.
- Exit code reflects success/failure for piping into other tooling.

Updated `/ceo/photos.md` and `/ceo/backlog.md` to reflect that the
stripper landed and to log a follow-up (Payload `beforeChange` hook
on Media for `contains-kids: true` items — turn the manual tool
